# Contributing to meshcore-llm-Lara

Thank you for your interest in contributing to meshcore-llm-Lara.
Please follow these guidelines to keep the project high-quality and easy to maintain.

## How to contribute

- Create a clean branch from `main` for your feature or fix.
- Open a pull request with a clear summary of the change.
- Keep commits small and focused.

## Reporting issues

If you find a bug or have a feature request:

1. Check existing issues to avoid duplicates.
2. Open a new issue with a concise title.
3. Include steps to reproduce, expected behavior, and actual behavior.

## Code style

- Use clear, descriptive variable and function names.
- Keep files formatted and readable.
- Document configuration options and user-facing behavior.

## Testing

- Add tests for new behavior whenever possible.
- Run existing tests before opening a pull request.

## Pull requests

- Describe the problem and the solution.
- Highlight any configuration changes or new requirements.
- Keep the pull request title and description concise.
